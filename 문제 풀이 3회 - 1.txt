1. 다음은 도메인에 대한 설명이다. 특징으로 알맞지 않은 것은?
1. 도메인별로 데이터 타입과 길이를 지정한다.
2. 각 엔터티 속성에 도메인을 할당한다.
3. 공통으로 발생하는 명사는 여러 개의 도메인으로 생성한다.
4. 속성을 명사로 분리한다.
답 : 3번
해설) 공통으로 발생하는 명사를 하나의 도메인으로 생성해야 한다.
도메인의 특징
 - 각 속성이 가질 수 있도록 허용된 값들의 집합이다.
 - 속성명과 도메인명이 반드시 동일할 필요는 없다.
 - 릴레이션에서 모든 속성의 도메인은 원자적이어야 한다.

계좌 마스터 테이블 [계좌번호(PK), 고객번호(FK), 예수금, 단말번호, 관리점 번호]

3. 단일 식별자, 인조 식별자

해설) 스스로 생성여부 : 내부식별자 VS 외부식별자
엔터티를 만들 때 이미 자동적으로 존재하느냐에 관한 문제
대표성여부 : 주식별자 VS 보조식별자
주요 메인키와 후보키

3. 다음은 주식별자에 대한 설명이다. 올바르지 않은 것은?
1. 어떤 업무에서 자주 이용되는 것을 주식별자로 한다.
2. 명칭, 내역 등과 같이 이름으로 기술되는 것들은 주식별자로 지정하지 않는다.
3. 계좌번호와 고객번호를 복합으로 주식별자로 구성할 경우 너무 많은 속성이 포함되지 않도록 한다.
4. 자주 변경되는 값을 주식별자로 지정해야 한다.
답 : 4번
해설) 주식별자의 특징
 - 주식별자에 의해 엔터티 내에 모든 인스턴스가 유일하게 구분되어야 한다.
 - 주식별자를 구성하는 속성의 수는 유일성을 만족하는 최소의 수가 되어야 한다.
 - 지정된 주식별자의 값은 자주 변하지 않는 것이어야 한다.
 - 주식별자가 지정되면 반드시 값이 들어와야 한다.

4. 아래 시나리오에서 엔터티로 가장 적절한 것은?
A 쇼핑몰에서 상품을 주문하려면 회원에 가입해야 한다. 회원가입 시에는 회원 ID, 
이름, 전화번호를 입력하고 주문을 할 때는 배송지 주소를 입력해야 한다.
1. 이름
2. 회원
3. A 쇼핑몰
4. 배송지 주소
답 : 2번
해설) 회원이라는 엔터티에 회원ID, 이름, 전화번호 속성이 있는 것이다.
엔터티는 2개 이상의 속성과 2개 이상의 인스턴스를 가진다.

5. 다음 중 주식별자의 특징으로 올바르지 않은 것은?
1. 유일성 : 엔터티 내에서 모든 인스턴스들은 유일해야 한다.
2. 최소성 : 속성의 수는 유일성을 만족해야 하고 최소의 수가 되어야 한다.
3. 불변성 : 식별자의 값은 변하지 않아야 한다.
4. 존재성 : 주식별자로 지정되면 데이터값이 존재하지 않을 수 있다.
답 : 4번
해설) 주식별자는 NULL 값을 가질 수가 없다.
주식별자 특징
 - 모든 인스턴스에서 유일해야 한다.
 - 주식별자의 속성 수는 최소여야한다.
 - 자주 변경되지 않는 것이어야 한다.
 - 반드시 값이 존재해야 한다.

6. 다음은 반정규화 절차에 대한 설명이다. 반정규화 수행 시에 뷰(VIEW) 혹은
클러스터링(Clustering) 기법을 적용해야 하는 단계는?
1. 반정규화 대상 확인
2. 다른 방법 결정
3. 정규화 여부 고려
4. 반정규화 실행
답 : 2번
해설) 반정규화 대상 확인 : 조인에 사용되는 테이블 및 대량의 데이터 처리, 
통계 처리 등을 확인한다.
다른 방법 결정 : 인덱스 조정을 통한 튜닝을 실시한다.
애플리케이션에 배열 처리를 수행한다.
뷰 혹은 클러스터링 기법을 적용한다.
반정규화 실행 : 테이블, 속성, 관계 등에 대한 반정규화를 수행한다.

7. 다음 중 ERD에 대한 설명으로 올바르지 않은 것은?
고객 테이블 [고객번호(PK), 고객명, 전화번호, 집주소]
주문 테이블 [주문번호(PK), 고객번호(FK), 제품번호(FK), 배송번호(FK), 주문일자]
제품 테이블 [제품번호(PK), 제품명
배송지정보 테이블 [배송번호(PK), 배송완료여부]
1. 고객번호가 없으면 주문을 할 수 없으며, 한 명의 고객은 여러 개의
주문을 할 수 있고 주문을 하지 않을 수도 있다.
2. 배송지는 여러 개의 주문을 배송할 수 있으며 배송지가 없을 수도 있다.
3. 주문과 제품의 관계에서 하나의 제품에는 하나의 주문만이 존재한다.
4. 주문일자는 고객이 주문을 한 일자로 배송이 완료된 후에 갱신되지 않는다.
답 : 3번
해설) ERD만 보고 답을 해야 하기 때문에 하나의 제품에 여러 개의 주문이 관계된다.

8. 다음 중 아래 테이블에 대한 관계를 설명한 것으로 가장 적절한 것은?
주문 테이블 [주문번호(PK), 회원번호, 주문금액],
상품 테이블 [상품번호(PK), 상품카테고리, 상품명, 상품가격]
1. 주문은 여러 개의 상품을 가질 수 있고, 상품은 하나의 주문에만 속할 수 있다.
2. 주문은 여러 개의 주문에 속할 수 있고, 주문은 하나의 상품만 가질 수 있다.
3. 주문 1개는 여러 개의 상품을 가질 수 있으며, 상품 1개는 여러 개의 주문에 속할 수 있다.
4. 주문은 상품을 하나도 안 가질 수 있다.
답 : 3번
해설) M : N 관계를 가지고 있는 ERD이다. 하나의 주문은 여러 개의 상품을 가질 수 있고
하나의 상품도 여러 개의 주문을 가질 수가 있다.

9. 다음 중 아래 테이블에 대한 이상현상을 설명한 것 중 가장 적절하지 안ㅇㅎ은 것은?
고객(PRIMARY KEY) [0001, 0002, 0003, 0004]
고객명[유비, 관우, 관우, 장비]
상품번호(PRIMARY KEY) [1000, 1000, 2000, 3000]
상품명[스마트폰, 스마트폰, 노트북, A카드]
가격[1000,1000,9000,5000]
1. 삽입 이상 : 상품을 주문하지 않은 고객의 정보를 삽입할 수 없다.
2. 갱신 이상 : 스마트폰의 정보를 업데이트할 경우 유비의 스마트폰만 업데이트하면 된다.
3. 갱신 이상 : 노트북의 가격을 업데이트할 경우 관우의 노트북만 업데이트하면 된다.
4. 삭제 이상 : 장비의 고객정보가 삭제되면 A카드 상품의 정보도 삭제된다.
답 : 2번
